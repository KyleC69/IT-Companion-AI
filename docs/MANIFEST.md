## Document Status Definitions

- **Active**: Current and maintained documentation
- **Draft**: Work in progress, not yet finalized
- **Deprecated**: Outdated documentation, kept for reference only
- **Archived**: Historical documentation, no longer relevant

## Version Control Guidelines

All documentation in this project follows semantic versioning (MAJOR.MINOR.PATCH):

- **MAJOR**: Breaking changes, complete restructuring, or significant conceptual shifts
- **MINOR**: New sections, substantial additions, or significant clarifications
- **PATCH**: Minor corrections, typos, formatting fixes, or small clarifications

## Change Request Process

1. Identify the document requiring changes
2. Update the document content
3. Update the version tracking header in the document
4. Update this manifest with the new version and date
5. Commit changes with a descriptive message

## Contributing to Documentation

When adding new documentation:

1. Include a version tracking header at the top of the document
2. Add an entry to this manifest
3. Follow the existing documentation style and format
4. Ensure all internal links are valid
5. Update the "Last Updated" date in the manifest

## Maintenance

This manifest should be reviewed and updated:
- Whenever a new document is added
- When a document is significantly updated
- When a document's status changes
- At least quarterly for general review
